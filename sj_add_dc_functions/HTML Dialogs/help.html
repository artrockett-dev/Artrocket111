3D text functions

Create an editable 3D text sub-component, and the attributes to configure its editing.
, text, line break character, font, alignment, italics, bold, fill, extrusion height, offset ...
, and a calculation attribute to recreate the geometry.
, The function must be called by an attribute named txt_ini

CREATE3DTEXT(newComponentName, text)

Parameter(s)

  • newComponentName : Name of the text subcomponent that will be created.
  • text : The text that will be modeled.

Function to edit 3D text.

EDIT3DTEXT(text, lineBreak, align, font, bold, italic, height, tolerance, zOffset, filled, extrusion, actualize, xOffset, yOffset)

Parameter(s)

  • text : The text that will be modeled.
  • lineBreak : Replacement character for line breaks.
  • align : left = 0, center = 1, right = 2
  • font : Writing font
  • bold : yes = 1, no = 0
  • italic : yes = 1, no = 0
  • height : Character height.
  • tolerance : Curve rounding tolerance 0.1 mm
  • zOffset : Offset the text on its Z axis.
  • filled : Outline only (no extrusion) = 0, Fill with outline = 1, Fill without outline = 2
  • extrusion : Text extrusion height in centimeters, only if filled > 0
  • actualize : Edit the text if update = 1, otherwise return unedited text
  • xOffset : Offset the text on its X axis.
  • yOffset : Offset the text on its Y axis.

Create a background sub component for the 3d text created with Create3dText.

Layers/Tags functions

Place the instance on the layer passed as a parameter.
Copies of a component obtained with the copies attribute are placed by Sketchup on layer0 or without tag, even if the original component is on another layer.
This function will force the copies to be placed on the layers of your choice.
If the layer / tag does not exist, the function generates it for you!

Parameter(s)

Very useful for lighting scenes.

Rotation functions

Allows incremental rotation of the component's existing transformation. The rotation takes place around the point of origin of the component targeted by the level attribute.
, Useful when called by the OnClic attribute.
, Example OnClic : Set("rot_result",SetRotation("z",90,1,1)
, Performs on a click, the rotation of the parent component around its Z axis by 90 degrees.

SETROTATION(axis, angle, lock, level)

Parameter(s)

  • axis : Axis letter, X, Y or Z.
  • angle : Value of the angle of rotation in degrees. Can be positive or negative.
  • lock : If lock = 1, the rotation takes place, otherwise it is not performed.
  • level : The rotation can be on the component that calls the function level = 0, or on its parent component level = 1, or the grand parent level = 2.

Allows incremental rotation of the component's existing transformation. The rotation takes place around the origin point of the component targeted by the offset attribute. The rotation is performed on the component targeted by the Level attribute. Useful when it is called by the OnClic attribute, OnClic
Example: Set ("rot_result", SetRotation ("z", 90,1,1,0)
Performs on a click, the rotation of the parent component on its Z axis of 90 degrees with the origin of the component calling the function at the center of rotation).

SETROTATIONOFFSET(axis, angle, lock, level, offset)

Parameter(s)

  • axis : Axis letter, X, Y or Z.
  • angle : Value of the angle of rotation in degrees. Can be positive or negative.
  • lock : If lock = 1, the rotation takes place, otherwise it is not performed.
  • level : The rotation can be on the component that calls the function level = 0, or on its parent component level = 1, or the grand parent level = 2.
  • offset : The center of the rotation can be the origin point of the calling component offset = 0, of the parent component offset = 1, or of the grand parent offset = 2. The value of the offset cannot be greater than the value of the level.

Create dynamic attributes to control a rotation when clicking on a child component.
, This function must be called from an attribute named "rot_ini".
, It creates 5 attributes: rot_x_angle, rot_y_angle, rot_z_angle, rot_view, rot_lock, then erases the formula of rot_ini and returns the message "Attributes created".
, If level = 0, the attributes are only created in the component, if level = 1, the attributes are created in the component and the parent component, with a return by formula of the values of the parent in the component.
, If level = 2 then ditto but on 3 level.

SETATTRIBUTROTATION(level)

Parameter(s)

  • level : 0 the component, 1 the parent, 2 the grandparent.

Parent/child attribute functions

Realizes the sum of the components / child groups on the attribute passed in parameter. Only child entities with this attribute are summed.

CHILDRENSUMATTRIBUT(attribute)

Parameter(s)

  • attribute : Name of the searched attribute.

Sum of the components / child groups on the attribute passed in parameters. For each child entities, if the conditional attribute / value comparison is true then the attribute is summed, otherwise 0.
, Example: Sum of the LenX attribute if the material is red.
, ChildrenSumSiAttribut ("LenX", "Material", "e", "Red")
, Example: Sum of the LenX attribute if the LenZ height greater than 3.
, ChildrenSumSiAttribute ("LenX", "LenZ", "s", 3)

CHILDRENSUMSIATTRIBUT(attribute, conditionalAttribute, operator, value)

Parameter(s)

  • attribute : Name of the attribute that will be summed, if the comparison is true.
  • conditionalAttribute : Name of the child attribute on which a comparison will be performed.
  • operator : Comparison operator: e for equal, ifor less, s for greater and d for different. By composition ie for less than or equal and se for greater or equal.
  • value : Comparison value. If it is a text only the operators e and d are authorized.

Returns the value or the path of a parent attribute according to the chosen mode.
It is convenient to replace the parent component name reference so that you can copy the subcomponent to another parent without error in calculations.

PARENTATTRIBUT(attribute, mode)

Parameter(s)

  • attribute : Name of the searched attribute.
  • mode : Two possible modes "path" or "value".

  • access_false : Index of the access property if the condition is false. In the list 0 => none, 1 => read only, 2 => entry, 3 => option list

Modifies the _access meta-attribute of attributes whose name begins with and according to a condition. Allows you to modify the appearance of the attribute in the option panel of dynamic components.

  • access_true : Index of the access property if the condition is true. In the list 0 => none, 1 => read only, 2 => entry, 3 => option list
  • access_false : Index of the access property if the condition is false. In the list 0 => none, 1 => read only, 2 => entry, 3 => option list

Modifies the _access meta-attribute of several attributes according to a common condition. Allows to modify the appearance of the attribute in the option panel of the dynamic components.

SWITCH_ACCESS_ATTRIBUTE_MULTI(condition, string_parameter_group)

  • string_parameter_group : Group of 3 parameters, the whole is in quotes. " attribute_name , access_true , access_false ". Each group is separated by a comma. At least one group is expected.

Boolean operations functions

Performs a boolean operation of sub-component volume 1 through subcomponent volume 2. The group, the result of the operation is renamed according to the parameter resultName.

GETBOOLEANCHILDREN(lock, operator, resultName, volume1Name, volume2Name, copyVolume1, copyVolume2)

Parameter(s)

  • lock : If 1 the operation is carried out otherwise frozen.
  • operator : Name of the boolean operation in English: union, substract, intersect.
  • resultName : Name of the group resulting from the operation.
  • volume1Name : Name of group or component definition or instance volume 1.
  • volume2Name : Name of group or component definition or instance volume 2.
  • copyVolume1 : If 1 works on a copy of volume 1, otherwise volume 1 will be destroyed by the operation.
  • copyVolume2 : If 1 works on a copy of volume 2, otherwise volume 2 will be destroyed by the operation.

Function that allows finding the result of a customizable mathematical function, which involves in its resolution the result of the same function for the previous integer.

Parameter(s)

  • n : The index of the suite to be calculated

  • f0 : The result for f(0)
